## 1. Project Overview

This is a **Competitor Analysis Dashboard** that transforms scraped competitor data (from Google Maps) into actionable business insights. The system follows a strict data pipeline architecture where all business logic resides in n8n, and the dashboard remains a **read-only** visualization layer.

### Vision
Menyediakan sistem terpusat untuk mengubah data hasil scraping kompetitor menjadi insight bisnis yang terstruktur, objektif, dan siap digunakan melalui dashboard analisis.

### Architecture Flow
```
Scraper
    ↓
n8n (Cleaning → Normalization → Enrichment → Scoring)
    ↓
Supabase PostgreSQL Database
    ↓
Next.js Dashboard (Read-Only)
```

## 3. Database Schema

### 3.1 `competitors` Table
Core competitor information from Google Maps scraping.

```sql
CREATE TABLE competitors (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
    place_id TEXT UNIQUE NOT NULL,
    name TEXT NOT NULL,
    main_category TEXT,
    categories TEXT[],
    rating NUMERIC(2, 1),
    reviews INT,
    website TEXT,
    address TEXT,
    is_spending_on_ads BOOLEAN DEFAULT FALSE,
    operational_status TEXT,
    market_query TEXT,
    last_updated TIMESTAMPTZ DEFAULT now()
);
```

### 3.2 `competitor_metrics` Table
Derived scores calculated by n8n pipeline.

```sql
CREATE TABLE competitor_metrics (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    competitor_id UUID UNIQUE REFERENCES competitors(id) ON DELETE CASCADE,
    reputation_score NUMERIC,
    digital_readiness_score NUMERIC,
    competition_level TEXT,
    risk_flag BOOLEAN
);
```

### 3.3 `swot_analysis` Table
SWOT analysis points per competitor.

```sql
CREATE TABLE swot_analysis (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    competitor_id UUID UNIQUE REFERENCES competitors(id) ON DELETE CASCADE,
    strength TEXT[],
    weakness TEXT[],
    opportunity TEXT[],
    threat TEXT[]
);
```

## 4. Scoring Logic (Implemented in n8n)

```javascript
// Reputation Score
Reputation Score = (rating * 0.6) + (normalized_reviews * 0.4)

// Digital Readiness Score
Digital Readiness Score =
    (website ? 0.4 : 0) +
    (is_spending_on_ads ? 0.3 : 0) +
    (featured_image ? 0.3 : 0)

// Competition Level
Competition Level = competitors_count / market_threshold
```

> ⚠️ **IMPORTANT**: All scoring logic must reside in n8n. The dashboard NEVER calculates scores.

### Supabase Integration
```jsx
// Client-side data fetching with RLS
import { useSupabaseClient, useUser } from '@supabase/auth-helpers-react'

const user = useUser()
const supabase = useSupabaseClient()

// Only query when authenticated
if (user) {
  const { data } = await supabase.from('competitors').select('*')
}
```

### Row Level Security (RLS)
```sql
-- Enable RLS
ALTER TABLE competitors ENABLE ROW LEVEL SECURITY;

-- Policy for project-based isolation
CREATE POLICY "Users can only access their project data" 
ON competitors FOR ALL 
USING (auth.uid() = user_id);
```

### TanStack Query Pattern
```typescript
function useCompetitors() {
  return useQuery({
    queryKey: ['competitors'],
    queryFn: async () => {
      const response = await fetch('/api/v1/competitors')
      return response.json()
    },
    staleTime: 1000 * 60, // 1 minute
  })
}
```

## 11. Key Constraints & Principles

### Architecture Rules
1. **n8n is the ONLY logic layer** - No business logic in frontend or backend
2. **Dashboard is READ-ONLY** - No edit/delete operations on competitors
3. **Backend is a thin BFF** - Only data access and ingestion proxy
4. **RLS enforces security** - Project-based data isolation at DB level

### Performance Requirements
- Dashboard load time: < 2 seconds
- API response time: < 500ms
